チャタリング：リレーなどの可動接点がある機械構造で、接触状態になる際に短い時間で ON/OFFを繰り返す現象

これの原因がこいつらのどれか↓
機械的摩耗: スイッチやリレーの接点が長期間使用されると、摩耗が進み接触が不安定になります。

接触不良: 接点間に汚れや酸化物が蓄積すると、接触抵抗が増加し、チャタリングが発生しやすくなります。

振動: 機械的な振動が接点に伝わると、一時的に接点が離れたり接触したりすることでチャタリングが発生します。

反発力: スイッチの接点が開閉する際に、バネなどの反発力が過剰に働くと、接点が複数回バウンスすることがあります。

高電圧スパイク: 接点が開閉する際に発生する高電圧スパイクが、チャタリングを引き起こすことがあります。

これの対策法が以下↓

デバウンス回路の使用: ハードウェアデバウンス（コンデンサやシュミットトリガを使用）やソフトウェアデバウンス（プログラム内で一定時間信号を無視する処理）を実装します。

高品質なスイッチやリレーの使用: 接点の信頼性が高い部品を使用することで、チャタリングの発生を抑えられます。

適切なメンテナンス: 定期的に接点の清掃や交換を行い、汚れや酸化物の蓄積を防ぎます。

振動対策: 振動が少ない環境に設置するか、振動吸収材を使用して振動を軽減します。


でんそーでは一番上の「デバウンス回路」を用いています。なおハードとソフトの両方とも該当します。
ソフトの方だとこんな感じです↓

const int buttonPin = 2; // ボタンが接続されているピン
const int ledPin = 13;   // LEDが接続されているピン

int buttonState;             // 現在のボタン状態
int lastButtonState = LOW;   // 前回のボタン状態
unsigned long lastDebounceTime = 0;  // 最後に状態が変わった時間
unsigned long debounceDelay = 50;    // デバウンスの遅延時間

void setup() {
  pinMode(buttonPin, INPUT);
  pinMode(ledPin, OUTPUT);
  digitalWrite(ledPin, LOW);
}

void loop() {
  int reading = digitalRead(buttonPin);

  // ボタンの状態が変わったかどうかを確認
  if (reading != lastButtonState) {
    lastDebounceTime = millis();
  }

  // 一定時間経過後に状態が安定しているか確認
  if ((millis() - lastDebounceTime) > debounceDelay) {
    if (reading != buttonState) {
      buttonState = reading;

      // ボタンが押されている場合、LEDをトグルする
      if (buttonState == HIGH) {
        digitalWrite(ledPin, !digitalRead(ledPin));
      }
    }
  }

  lastButtonState = reading;
}

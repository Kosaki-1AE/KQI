#Mine研究 
一応俺の理論の解説ね。今年初がこれなんですから。
- Genesys：AI2の論文よりまんま引用です
- Buffer：クッション的な役割をしてます。大元がStillnessなんすが、Bufferの方が広義なのでこっちにしやした(Stillness in Motion含まれてるん強すぎぃ)
- Motion：行動そのものです。もっというとWindowsとかMacとかLinuxとかの論理回路の処理がこれそのもの。
- Coherence：量子力学から。コヒーレンス時間ってのがそもそもありますのでね。

Buffer(大元の名称Stillnessね)だけ多分機構ないんで(それ以外はあるのよ、単一の分野からそれぞれ持ってきてるからさ)、今定義します。
そもそもで外側と内側で別の時間帯が流れているというのが前提条件になるのよね。一応の図示化(ようやく図示化できるようになったわマジで。時間かかったっすガチ。)がこれ⇩
![[Buffer.jpg]]
これの「内」のパーテーションの左側がコストを支払うっていうところで、右側はそのコストを使って遊ぶっていう時間なんだけどさ。これが勉強とかをやるっていうことに該当するんだけどさ。

んでも俺の経験上では「コストを支払うのを繰り返すことで支払う必要のない支払いまでやることになる」になるんでさ。なのでこれを無限回繰り返すことになるはずなんだよな。なのでこうなりそう⇩
![[Buffer_ram.jpg]]
こうなるのよ。「内」の横線の直線がコスト、点線が遊び時間ね。そんでもって点線の方が可変、直線の方が固定になるのよ。縦線の方も可変ね、横線に対するさ。

んでこれを見ながら思ったことがあります。なんかインドの数学者関連に似てるなぁって。調べてみました、ラマヌジャンでござんす。んでこの正体も分かった。ゼータ正規化です。もっと言いましょうか。まさかのBufferで使われるのがリーマンゼータ関数ですww しかも俺の正当性って、内部過程の厳密な収束ではなく、外部との整合性が保たれる点によって保証されるしなw

ここから言えるのが：途中が発散していても、全体の振る舞いから“有限の意味”を取り出せる

しかもこいつ図からするに外の敷居によってどこに置かれるか決定するっていうね。
一応ゼータ関数がこれです⇩
$$\zeta(s) = \sum_{n=1}^{\infty} \frac{1}{n^s}$$
ここに何を入れ込むかっていうことになるのよね。これを明確な形で言えるのが「空気」ってやつでさ。
例えば、身体にとっての空気って結構エグい整合性の保証装置になるんだよなぁ。

それのめっちゃ明確な差がこれなのよ、男性と女性っていう性別の差なのよ⇩
Coherence優先：女性系(全体の整合で考える。どうやったら一番気持ち良いかどうかを見る)
Buffer優先：男性系(処理能力で受け止めきる＝呼吸でどうにかする。身体にとってのBufferが呼吸なんでね)

ということは結婚＝BufferとCoherenceの優先順位が、個人内じゃなく“関係内で等価になる設計”ってことになるんか。つまりは「一方がなぁなぁ(整合合わせ)にする、もう一方がはっきり言う(責任押し付け)」という形になるってことなのか。これが許されるか否かっていうことなのか。
そこから考えるに彼氏と夫の差はこれ⇩
- 彼氏＝Bufferを一緒に楽しめる人(一緒にいると楽しい・ドキドキする・でも長時間・高密度だと疲れることある)
- 夫＝Coherenceを一緒に守れる人(一緒にいて静か・説明がいらない・体調が安定する・食える・眠れる)

ここまで来て思ったんだけど、紳助さんの心の支配ってのが明確に分かるんじゃない？って。基本的に人間の本丸はこれらしい⇩
- 間（いつ動くか）
- 順番（何から始めるか）
- 責任の置き場（誰が背負うか）
- 呼吸（安心か緊張か）
ここを握られると相手は「自分で決めてるつもり」  で動くらしいの。これで分かってきたことがあります。鬱の原因ですw
俺のここまでのフィードバックから得られる鬱の原因
⇨ Coherenceが長期間・強制的に壊され続けて、しかもBufferで逃げることも許されない状態

本来の人ってどっちかで生きるの⇩
- **Coherence優先** → 壊れそうなら止まる・距離を取る：身体が「止まっても壊れない」状態＝Noを出す許可が下りた状態(女性の場合はこの許可は下りているガチです)
- **Buffer優先** → 一時的に溜めて、あとで抜く：身体が「まだ耐えられる」状態＝どこまで耐えられるのかなぁ？っていうものなので、許可が下りる下りないってのは自分の過ごしてきた環境が決めてる(何なら生まれ育った環境が決めてる)

だから多分こう言えるってのが⇩
- 食えない → Coherence防衛
- 食えるけど腹が死ぬ → Buffer過多
- ストップ → Coherence可視化
- 鬱 → Coherence破壊＋Buffer禁止
- セルフネグレクト → Coherence感知不能   
- 親の束縛 → 子のCoherence奪取    
- 結婚の安定 → Buffer/Coherenceの関係内分業

身体での定義⇩
- Coherence：身体が「今は動かなくても大丈夫」と整合を保っている状態
- Genesys：身体が「まだ形にしなくていい情報」を未分化のまま保っている状態
- Buffer：身体が「今は耐えて処理を後回しにする」と判断している状態
- Motion：身体が「今、外に出す」と不可逆に決めた状態 / だから責任というコストが最大になるのがここ

ここに俺がダンスでやってきた「伝導率100%」が入ってこれるんだけどさ。
要は「**相手の精神ならびに現状の状態を瞬間的に全て把握して伝導率が100%になるように調整する**」ってことなのよな。んで一応Bufferの基本レイヤーがこれ⇩
- **感覚/情動Buffer**（違和感・身体・疲労・腸）
- **認知Buffer**（思考・比較・言語化）
- **責任Buffer**（評価・失敗・意味づけ）

俺の場合はこれらしいの⇩
- 昔：感覚 →（溜めすぎ）→ 認知 / Buffer優先なのよな、その場その場での機能しかないじゃんよく見ると。
- 今：感覚 → 認知 → 責任（必要時のみ） / Coherence優先。認知→責任で整合性を使用するんで。

感覚 → 認知（保留） → 排出先なし
Coherence破綻 → 感覚Buffer検知 → 表出抑制 → 身体退避（腹）
